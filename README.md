## 📖 教科書で押さえるべき重要ポイント（学習メモ）

実習を始める前に，教科書（実教『最新情報Ⅰ』94〜99ページ）に掲載されている情報デザインの基本概念を確認しましょう．

### 1．情報の分類・整理（LATCHの原則）
すべての情報を分類・整理するための代表的な5つの観点（それぞれの頭文字をとって **LATCH（ラッチ）** と呼びます）です．提唱者はアメリカのデザイナーであるリチャード・ソール・ワーマンです．

- **L**ocation（位置）：建物のフロアマップや日本地図，路線図など，物理的な「場所」や「空間」で整理する方法．
- **A**lphabet（アルファベット）：国語辞典の五十音順やスマートフォンのアドレス帳など，文字や名前の「規則的な並び順」で整理する方法．
- **T**ime（時間）：歴史の年表やスケジュール帳，電車の時刻表など，情報の「時間軸や前後関係（時系列）」に沿って整理する方法．
- **C**ategory（分野）：図書館の本棚やECサイトの商品ジャンルなど，情報の「類似性や共通点」でグループ分けして整理する方法．
- **H**ierarchy（階層）：重要度によるランキングや，商品の価格順など，特定の基準による「優先度やレベルの差（大小関係）」で整理する方法．

💡 **【今回の実習でのLATCHの活用例】**
今回の未整理な文章を構造化するにあたり，LATCHの以下の要素を意識して情報を整理しましょう．

1. **Category (カテゴリ) による分類：**
   時系列や持ち物が入り乱れている文章から，**「イベント概要（日時や場所）」「持ち物・服装」「当日のプログラム（スケジュール）」「問い合わせ窓口」** といった情報の共通点を見つけてグループ分けし，それぞれに見出しの名前（カテゴリ名）を設定します．
2. **Hierarchy (階層) による親子関係の定義：**
   情報の重要度や大きさを考え，全体のタイトルには `<h1>` を，自分で分類した各グループの見出しには `<h2>` を適用し，Webページ全体の論理構造（アウトライン）を設計します．
3. **Time (時間) に沿った整頓：**
   文章の中にバラバラに書かれている当日のタイムスケジュールを，時間の順序（時系列）通りに正しく並び替えて `<table>` に整理します．

### 2．レポートの基本構成とHTML構造化
WebページのHTMLタグの階層構造は，学校や大学で書く **「レポート・論文の構成」** とまったく同じ考え方です．
- **レポートの構成：** タイトル ➔ 章 ➔ 節 ➔ 項 ➔ 本文
- **HTMLの構成：** `<h1>` ➔ `<h2>` ➔ `<h3>` ➔ `<h4>` ➔ `<p>`
見出しタグを正しい順番（`h1` ➔ `h2` ➔ `h3`）で使うことで，論理的に整理された読みやすい構成になります．

### 3．情報バリアフリーとアクセシビリティ
なぜ，単に文字の大きさや色を変更（ビジュアルの調整）するだけでなく，正しい「タグ（HTML）」を選んで文書構造を定義する必要があるのでしょうか．ここには「情報のユニバーサルデザイン」としての重要な目的があります．

#### 情報バリアフリー（認知的・環境的バリアの解消）
情報バリアフリーとは，年齢や障害の有無，あるいは状況的な制限（スマホの画面が反射して見づらい，母国語ではないため複雑な日本語が読みづらい等）に関わらず，**すべての人が情報にアクセスし，理解しやすいように障害（バリア）を取り除くこと**です．
- **今回の実習での具体例：** 
  もしWebページが今回の初期状態のようなダラダラとした平文のままだと，文章を読むのが苦手な人や，外国人（日本語学習者），あるいは急いでいて時間がない人にとって「重要な情報（持ち物やスケジュール）」を見落とすバリアになります．箇条書き（`<ul>`）や表（`<table>`）を用いて情報を整理することは，こうした読みにくさを解消する重要な情報バリアフリーです．

#### Webアクセシビリティ（身体的・技術的バリアの解消）
アクセシビリティとは，特に高齢者や障害のある人が，支援技術（アシスティブ・テクノロジー）を用いて**Webページを利用可能（Accessible）にすること**です．
- **今回の実習での具体例（音声読み上げへの対応）：**
  視覚障害のある人が使用する「音声読み上げソフト（スクリーンリーダー）」は，HTMLの「タグ」を頼りに文書の構造を理解します．見出しタグ（`h1`, `h2`）が正しく設定されていれば，ユーザーはキーボード操作だけで「次の見出しへジャンプ」することができ，不要な挨拶文や無関係なノイズ情報をスキップして，必要な「スケジュール」や「問い合わせ先」だけを素早く読み上げさせることができます．正しいタグを使わず，見た目（CSS）だけで文字を大きくしている場合，このジャンプ機能が使えず，最初から最後まで延々と読み上げを聞き続けなければならないという大きなバリアが発生してしまいます．

見た目の変更（CSS）を行う前に，まず正しいタグ（HTML）で文書の論理構造を定義することが，情報バリアフリーおよびアクセシビリティの第一歩です．

## 🛠 構造化の要件（ミッションクリア基準）

以下の要件を満たすように `index.html` の `<body>` タグ内を書き換えてください．

1. **不要な情報の除外（取捨選択）：**
   - 文章中から，一般参加者にとって不要な情報（ノイズ情報：自治会役員向けの集合時間，懇親会・反省会の案内，無関係な思い出話など）を取捨選択し，削除するか目立たないように整理してください．
2. **見出しの階層化（Hierarchy / Category）：**
   - 最も重要な全体のタイトルには `<h1>` を使います．
   - 情報を自分で分類した4つのグループ（「概要」「持ち物」「スケジュール」「問い合わせ」など）の見出しには，**自分で考えた適切な見出しの言葉**をつけ，`<h2>` を使用します．
3. **箇条書きリストの作成（Category）：**
   - 文章中に散らばっている「持ち物の情報」を抜き出し，順序なしリストタグ（`<ul>` と `<li>`）を使って正しく構造化します（計5つのアイテムがあります）．（`<br>` による改行は使用しません）
4. **表の作成（Time）：**
   - 文章中のタイムスケジュールのデータを，時間の順番（時系列）通りに整理し，表タグ（`<table>`，`<tr>`，`<th>`，`<td>`）を使って整理します（ヘッダー行＋データ4行の計5行構成）．（`<br>` による改行や空白スペースによる調整は使用しません）
   - 表の1行目（ヘッダー）には `<th>` を，データ行には `<td>` を使用してください（「時間」「内容」「担当者」の3列のヘッダーを作成します）．

## 💾 提出と自動評価の確認手順

### Step 1. AIとのチャット履歴の保存
AIと対話した場合は，チャットパネルの右上「…」から **「チャットのエクスポート」** を選択し，`copilot-chat-history.json` という名前でこのフォルダ直下に保存します．（対話していない場合も，ファイルが空またはファイル自体がなくても自動評価は実行されますが，評価対象となります）

### Step 2. Gitによるコミットとプッシュ
左側のソース管理パネル（ブランチアイコン）を開き，以下の手順を実行します．
1. 変更したファイル（`index.html`，`copilot-chat-history.json`）の横にある **「＋」（ステージ）** をクリックします．
2. メッセージ入力欄に **`feat: structure index.html`** と入力します．
3. **「コミット」** ボタンをクリックします．
4. **「変更の同期（プッシュ）」** をクリックして，GitHub（クラウド）に送信します．

### Step 3. 自動評価（Issue）の確認
1. ブラウザで自分のGitHubリポジトリページを開きます．
2. 画面上部にある **「Issues」** タブをクリックします．
3. **「【自動評価】第5回課題 ルーブリックフィードバック」** というIssueが届いているか確認します（プッシュから約1分で届きます）．
4. 中身を開き，すべて🟢（合格）になっていればミッション完了です！
5. 🔴（要修正）がある場合は，アドバイスを読んでCodespacesでコードを修正し，再度 **Step 2（コミット＆プッシュ）** を実行してください．Issueが自動で再評価され，内容が上書き更新されます．
